# WO-0198 -- Explorer Lane Schema, Content Template, and Hero-Asset Brief System

**Status:** COMPLETED
**Priority:** P0
**Created:** 2026-04-22
**Completed:** 2026-04-22

## Objective

Define the structural schema for the interactive explorer so each lane has a consistent content model, visual-asset brief format, and evidence pattern before implementation begins.

## Context

The explorer is intended to become the centerpiece of the `Home` cluster, but it cannot be designed as a vague interaction toy. Each lane needs the same structural payload:

- hero graphic
- exact use-case header
- short summary
- why UM fits
- what actually travels
- expandable deeper detail
- relevant proof/spec links

This work order exists to define that template and the lane inventory before content writing and UI work.

## Scope

In scope:

1. Define the lane schema for explorer content.
2. Define the standard layout/content payload per lane.
3. Define the hero-graphic brief format.
4. Define the first lane inventory and ordering.
5. Define the relationship between explorer lanes and supporting audience pages.

Out of scope:

- writing every lane’s finished content
- building the explorer UI
- producing final graphics

## Deliverables

- Explorer lane content schema.
- Hero-asset brief template.
- Ordered lane inventory for first implementation.
