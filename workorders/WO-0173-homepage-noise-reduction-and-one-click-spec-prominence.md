# WO-0173 -- Homepage Noise Reduction and One-Click Spec Prominence

**Status:** COMPLETED
**Priority:** P0
**Created:** 2026-04-12
**Completed:** 2026-04-13

## Objective

Tune the homepage copy, hierarchy, and calls to action so a first-time visitor can understand Universal Manifest quickly and immediately see how to reach the latest spec.

## Context

The two-page MVP only works if the homepage does its job with far less noise than the current broader public site. The user requirement is explicit:

- no overwhelming message density,
- no unnecessary technical framing,
- obvious path to the latest spec,
- comprehension first, taxonomy second.

This work order exists separately from WO-0172 so the public route contraction and the messaging-quality pass can be tracked independently.

## Scope

In scope:

1. Simplify homepage hero and supporting sections for zero-context visitors.
2. Reduce message density and eliminate secondary material that competes with the core story.
3. Make the `/spec/latest/` call to action unmistakable.
4. Verify that the homepage leads with the problem UM solves rather than internal site structure.
5. Tune section order so comprehension comes before route branching.

Out of scope:

- Broader route reintegration.
- Internal component extraction.
- Tool-shell unification.

## Deliverables

- Refined homepage messaging hierarchy.
- Clear one-click path to the latest spec.
- Reduced homepage section count and noise.
- Verification notes for comprehension and CTA clarity.

## Dependencies

- WO-0172 defines the minimal MVP shell that this tuning pass refines.

## Acceptance Criteria

- [x] The homepage reads clearly for a visitor with no prior context.
- [x] The latest spec call to action is visually obvious and not buried.
- [x] The homepage no longer feels overloaded with route options or technical detail.
- [x] The message hierarchy prioritizes problem framing, value, and next action.
