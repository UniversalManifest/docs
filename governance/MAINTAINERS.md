## Maintainer Responsibilities

Maintainers are responsible for:

1. **Code and Specification Review**
   - Review pull requests for technical quality
   - Ensure specification consistency and clarity
   - Validate conformance test coverage

2. **Community Engagement**
   - Respond to issues and discussions
   - Guide new contributors
   - Facilitate RFC review process

3. **Technical Stewardship**
   - Participate in Technical Steering Committee decisions
   - Guide architectural direction
   - Ensure interoperability and adoption success

4. **Quality Assurance**
   - Uphold the done-done framework
   - Validate release readiness
   - Maintain documentation quality

## Becoming a Maintainer

Maintainership is earned through sustained high-quality contributions. The typical path:

1. **Consistent Contributions** (6+ months)
   - Regular pull requests
   - Thoughtful issue triage
   - Community participation

2. **Technical Depth**
   - Deep understanding of specification goals
   - Strong technical judgment
   - Attention to backward compatibility and interoperability

3. **Collaboration Skills**
   - Constructive code review
   - Clear communication
   - Consensus-building approach

4. **Nomination Process**
   - Existing maintainer nominates candidate
   - TSC reviews contribution history
   - Majority vote required for approval
